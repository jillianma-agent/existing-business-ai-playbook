# Deployment Configuration

Technical details for monday-code CDN deployment.

## CDN Deployment Command

```bash
mapps code:push -c -d dist -a <APP_ID> --force
```

| Flag | Purpose |
|---|---|
| `-c` | Client-side / CDN deployment |
| `-d dist` | Directory containing built static assets |
| `-a <APP_ID>` | Target monday app ID |
| `--force` | Skip interactive confirmation prompts |

After deployment, clean up the artifact: `rm -f dist.zip`

## Package.json Scripts

Every site's `package.json` should include:

```json
{
  "scripts": {
    "dev": "vite",
    "build": "rm -rf dist && tsc && vite build",
    "preview": "vite preview",
    "deploy": "mapps code:push -c -d dist -a ${MONDAY_APP_ID:?} --force && rm -f dist.zip"
  }
}
```

The `deploy` script uses `${MONDAY_APP_ID:?}` which fails fast if the env var is not set.

## Required Files

| File | Location | Purpose |
|---|---|---|
| `.mondaycoderc` | Project root | Specifies Node.js runtime |
| `manifest.json` | Project root | App manifest with features |
| `index.js` | Frontend root | CDN entry point (required, can be comment-only) |
| `vite.config.ts` | Frontend root | Must include `define: { global: "globalThis" }` |

### .mondaycoderc

```json
{
  "runtime": "nodejs22.x"
}
```

Supported: `nodejs20.x`, `nodejs22.x`, `nodejs24.x`

### manifest.json

```json
{
  "name": "Site: {site-slug}",
  "description": "{description}",
  "features": [
    {
      "type": "AppFeatureBoardView",
      "name": "{site-slug}",
      "settings": {}
    }
  ],
  "oauthScopes": []
}
```

### vite.config.ts

```typescript
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig({
  plugins: [react()],
  define: {
    global: "globalThis",
  },
  build: {
    outDir: "dist",
  },
  server: {
    port: 3000,
  },
});
```

The `global: "globalThis"` polyfill is required for the monday SDK to work in browser context.

## CDN URL

After deployment, the site is accessible at:

```
https://<hash>.cdn2.monday.app/
```

The hash is generated by the platform based on the app and version. It is permanent for the app.

## Verification

```bash
curl -s -o /dev/null -w "%{http_code}" https://<hash>.cdn2.monday.app/
```

Expect HTTP 200 for a successful deployment.

## CLI Installation

```bash
npm install -g @mondaycom/apps-cli
mapps init
```

`mapps init` authenticates with the monday.com API via browser OAuth.

## Account Limits

| Resource | Limit |
|---|---|
| Private apps with monday-code | 5 per account |
| Pages per app (via routing) | Unlimited |
| CDN bandwidth | No documented limit |
| File size | Standard CDN limits apply |

If the user needs more than 5 independent sites, options:
1. Group more pages into fewer apps via routing
2. Make apps public (removes the 5-app limit but has marketplace visibility implications)
3. Host some sites externally

## Troubleshooting

### "mapps: command not found"
```bash
npm install -g @mondaycom/apps-cli
```

### "MONDAY_APP_ID is not set"
```bash
export MONDAY_APP_ID=<app-id-from-sites-json>
```

### Deploy succeeds but CDN returns 404
- Ensure `index.html` exists in the `dist/` directory
- Ensure `index.js` exists at the frontend root
- Check that `vite build` completed without errors

### CDN URL unknown
- The CDN URL is returned in the `create_app_feature` response
- It can also be found in the Developer Center under the app's feature settings
- Use MCP `get_app_features` to retrieve it programmatically
